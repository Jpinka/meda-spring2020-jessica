Just Learn the Basics for Now

So how do I code?! Which language should I learn first?! Where do I start?!

Well, we need to know the difference between a programming language and a computer language. A programming language uses algorithms in order to communicate with the computer to preform a specific task. A computer language, like HTML, communicates with the computer directly. AT least, that is the jist of it. There are two links above for more information surrounding the topic. Let's clear up a common misconception, there is no right or wrong language to learn first. However, I do recommended starting with HTML, a computer markup language that uses tags to strcture and define elements within a document. FUN FACT: Every webpage contains HTML. A starting computer language, that is very basic and a great starting point for absoliute beginners who are starting out. You will quickly figure out whether or not Web Development is for you based on your experience with HTML. It reminds me of a strctured template and the wed developer plugs in the content and uses differents elements to communicate witht the computer.

Another "FUN FACT" Craigslist was made primarly with HTML. If you click on the provided link you can see just how basic an HTML webpage could look like. Would you all like to see super basic HTML on paper? I wrote somehting out for you look below. What you are lookig at is very basic but you can do a lot with it That is enough code to write a poem with a title. The DOCTYPE text is a code that tells the computer you are writing am HTML document so that we browsers can translate them to plain english. the brackets attatched to each word are called tags. The body stands refers to the body of the document, h1 stands for 1st header. That leaves us with the p tag, which identifies a paragraph within the body. When you put that code in. You get the results as shown below the. Becasue h1 is a header the text is automatically bigger than the rest of the documents.

Second, on my list is CSS this is a computer mark up language similar to HTML and they usually work hand to hand. Also called the sister language to HTML. HTML gives a webpage its content and CSS is responsible for the design of a website. Think of Craigslist, it is mostly HTML. Simple black and white, very little color aside from images posted by the user and the color of links. If we added color inside the headings and body of each categories, that would be CSS, also known as the sister language to HTML.It is also an easy language to learn if you have previously completed HTML.



That leaves us with JavaScript. Javascript is a programming language and it is responsible for the actions and movements that are viewed on a webpage. Think of a basic object, I'll pick a ball. Picture the ball moving from the left of your screen, to the right then repeat itself. That is JavaScript. That is also the starting basics for some tennis games if you can picture that. Yes, you can also build basic games with JavaScript. J.S. is a fun and not super difficult language to learn. However, it is more challenging and complex than HTML or CSS. What do you think will hapen if we put all of the languages to create mini robots? Look down. After you gain experience writing code you start to see the world in code. Example: We see these two robots. the HTML is the plan box itself, the strcture of each robot. The CSS is the color added and it is how we identify what the box strctures are. Finally, Javascript is the reason our robots are walking and have lightup eyes.


If you can learn these three languages, consider this, a future congratulations!!! You will become a beginner web developer.

Kahn Academy as a Free Resource

Here is what you should do after leaving my web page, sign up for Kahn Academy. It is free website and it will get you started. I recommended K.A. as a starting point because the instructional videos are entertaining, well made, and beginner friendly. There is no limit on how much you can learn or how often you can utilize their resources, believe me it helped a lot. I especially like their open source community of students. Open source is a common term in the programming industry which means that anyone can assess your code and view it. The purpose is to learn, then as you grow. Before I move on, I wanted to mention that every instructional video comes with a printable transcript and chapter summaries. But hey do not take my word for it, sign up and see for yourself. If you dsecide you dont like developing or coding. Learn somehting else.

What happens after you learn HTML, CSS, and J.S.? Take a week off and do it again! I recommend it, anyway. Free Mobile Apps

Raise your hand if you knew you could code using mobile apps? I didn't know that starting out! Luckily, I found out. So, there are many mobile apps available via play store on your Android or Apple smart phone that will aid you in learning to code. Big question? Can you really learn to code using a smart phone? Yes, and no, everyone is different, keep that in mind. I would say you can potentially learn a new language. More or less become familiar with that language. In most cases you have to build projects to actually claim that you have learned that particular language. I recommend SoloLearn and Encode

SoloLearn is surprisingly well developed and well managed. Words cannot describe how amazed I was learning code in my space time. I encourage you to download SoloLearn first look around and see the many languages their app supports. Like, Kahn Academy, they too have an open source community and multiple ways to seek help in needed. Encode should be used to challange yourself, they offer many different exercises for you to complete in order to test yourself on your knowledge. It gives you a sense of understanding where you are at. It also keeps your mind busy and thinking about code when you do not have the time to dedicate to sitting in front of a computer.

Any other computer or programing language is optional, it depends on the metaphorical path you want to travel on. Just dont travel the bridge to noware. It can also depend on what it is you are tryig to build. You can build web pages like me, or move on to mobile apps, and even video games. Which ever path you choose it should be based on your intreasts. Use Web Development as a starting point to see if you even like it. In other words, if you hate or cant comprehend HTML, CSS or JavaScript then maybe coding is not foo you. Each language you atempt to learn will take you on a different path. Speaking of pathways.
